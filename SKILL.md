# Hairstyle Transfer Skill

## 核心原則

本 Skill 用於把目標人物照片中的髮型與髮色，替換成另一張髮型範例圖中的效果。

**最重要原則：`REFERENCE_0` 是唯一人物身份來源。**

- `REFERENCE_0`：目標人物圖像，唯一允許作為人物臉部、身份、表情、五官、膚質與氣質參考的圖片。
- `REFERENCE_1`：髮型範例圖，只允許用來分析髮型與髮色，不得作為人物臉部、五官、妝容、氣質、年齡感或身份參考。

本 Skill 不是把兩張人像直接混合，而是分成兩步：

1. 先從 `REFERENCE_1` 提取髮型文字描述，生成 `scope` 與 `hair_color`。
2. 再把 `scope` 與 `hair_color` 填入 JSON 工作流，以 `REFERENCE_0` 為唯一基礎進行局部換髮。

這樣可以最大限度避免髮型範例圖中的臉、妝容與身份特徵污染目標人物。

## 輸出目標

最終只輸出 **一張圖片**：

- 左側：`REFERENCE_0` 中目標人物換髮後的正面效果。
- 右側：同一人物換上同款髮型後的背視效果，用於展示髮型背面的長度、層次、髮量、捲度、髮尾與髮色。
- 左上角：加入髮型名稱或簡短髮型描述。

文字規則：

- 使用繁體中文。
- 文字顏色固定為 `#721D24`。
- 不需要英文副標。
- 不添加 Logo、水印、社交媒體數字或無關 UI 元素。

## 觸發條件

當用戶提出以下類型需求時，啟用本 Skill：

- 「幫我把這個人的髮型換成另一張圖的髮型」
- 「用圖二的髮型替換圖一人物的髮型」
- 「根據範例圖換髮」
- 「只改髮型和髮色，其他保持不變」
- 「生成正面和背面拼在一起的換髮展示圖」

## 圖片輸入判斷流程

### 情況 A：用戶已上傳兩張圖片

若當前任務中已存在兩張圖片：

- 第一張圖片視為 `REFERENCE_0`：目標人物頭像。
- 第二張圖片視為 `REFERENCE_1`：欲置換的髮型範例。

直接進入「髮型文字提取」與「JSON 編譯」流程。

### 情況 B：用戶只上傳了一張圖片

若當前任務中只有一張圖片：

- 將該圖片視為 `REFERENCE_0`，即目標人物頭像。
- 暫停生成。
- 提示用戶補充上傳第二張圖片，即欲置換的髮型範例。

建議回覆：

> 我已收到目標人物頭像。請再上傳一張包含你想置換髮型的範例圖。我會只分析第二張圖的髮型與髮色，不把第二張圖當作人物臉部參考。

### 情況 C：用戶沒有上傳圖片

若當前任務中沒有任何圖片：

- 暫停生成。
- 提示用戶先上傳目標人物頭像。

建議回覆：

> 請先上傳一張目標人物頭像。收到後，我會再請你上傳一張髮型範例圖。

### 情況 D：收到第一張圖後的後續提示

當用戶補充第一張目標人物圖像後，繼續提示：

> 目標人物頭像已收到。請再上傳一張包含想要置換髮型的範例圖。

## 工作流總覽

### 第一步：只分析 `REFERENCE_1` 的髮型，不生成圖片

從 `REFERENCE_1` 中提取以下內容：

- 髮型名稱：若能準確命名，使用常見中文名稱；若無法命名，給出簡短描述。
- 分線：中分、偏分、無明顯分線、瀏海等。
- 長度：下巴、鎖骨、胸部上緣、胸下、肩胛骨、腰部等。
- 捲度：直髮、內扣、S 型波浪、大波浪、水波紋、羊毛捲、螺旋捲等。
- 層次：齊髮尾、低層次、高層次、臉周層次、髮尾層次等。
- 髮量與蓬鬆度：貼頭皮、自然蓬鬆、空氣感、豐厚等。
- 臉周髮絲：貼臉碎髮、八字瀏海、face-framing layers 等。
- 髮色：主色、冷暖調、明度、飽和度、反光色、霧感等。
- 質感：柔順、毛流清晰、細碎飛髮、沙龍光澤、霧面質感等。

注意：

- 不得描述 `REFERENCE_1` 中人物的臉型、五官、妝容、皮膚、年齡、身份氣質。
- 若必須提到臉周髮絲，只描述頭髮如何包覆臉部，不描述範例人物的臉。

### 第二步：把髮型分析結果編譯成 `scope` 與 `hair_color`

生成兩個欄位：

```json
{
  "scope": "僅髮型與髮色：……",
  "hair_color": "……"
}
```

`scope` 應包含：

- 髮型名稱或描述。
- 分線。
- 髮長。
- 捲度或直髮形態。
- 層次。
- 臉周髮絲。
- 髮尾走向。
- 髮量與蓬鬆度。
- 整體風格。
- 明確避免與範例不符的髮型，例如避免羊毛捲、避免螺旋捲、避免厚重齊髮尾等。

`hair_color` 應包含：

- 主色。
- 冷暖調。
- 明度與飽和度。
- 反光色。
- 質感，如霧面、柔順光澤、沙龍光澤等。
- 明確避免錯誤髮色。

### 第三步：套入 JSON 工作流

把第二步得到的 `scope` 與 `hair_color` 填入以下 JSON 結構。

### 第四步：最終生成策略

最終生成時，遵守以下策略：

1. 把 `REFERENCE_0` 作為唯一人物身份來源。
2. 不把 `REFERENCE_1` 傳入最終生成階段作為人物視覺參考；如果系統必須保留附件上下文，明確聲明 `REFERENCE_1` 僅作髮型分析，不能影響臉。
3. 允許內部臨時推理正面效果與背面效果，但最終只輸出一張左右拼接圖。
4. 左側正面圖必須優先確保人物臉部一致性。
5. 右側背視圖可以不露臉，以降低生成新人物臉的風險。

## 髮型命名規則

若能從範例圖判斷髮型，優先使用以下命名方式：

- 韓系空氣感中分鮑伯
- 韓系高層次 S 型大波浪
- 韓系女神水波紋長髮
- 法式大波浪
- 法式慵懶半盤髮
- 日系半盤髮
- 鎖骨髮
- 中長髮內扣
- 八字瀏海長捲髮

若不好命名，使用描述式命名，例如：

- 中分低層次柔霧棕長捲髮
- 胸下長度自然水波紋捲髮
- 低位鬆散半盤髮
- 冷調奶茶棕短鮑伯

## 失敗修正策略

若生成結果人物不像 `REFERENCE_0`：

1. 降低髮型相似度要求，優先恢復人物臉部一致性。
2. 明確要求：正面圖只在原圖頭髮區域局部重繪，不重繪臉部。
3. 取消或減弱臉周遮擋五官的髮絲。
4. 將最終提示詞改為「先生成正面局部換髮效果，再把同款髮型背視圖拼到右側」，但最終仍只返回一張拼接圖。

若髮型不像 `REFERENCE_1`：

1. 重新分析 `REFERENCE_1` 的髮型，補充髮長、分線、捲度、層次與髮尾走向。
2. 只把新增髮型文字補入 `scope`，不要讓 `REFERENCE_1` 直接影響人物臉部。
3. 可要求再次參考 `REFERENCE_1` 的髮型結構，但仍禁止參考其臉部與妝容。
