# Hairstyle Transfer Skill

[English](README.md) | [Português](README.pt.md) | [中文](README.zh-Hant.md) | 日本語

`hairstyle-transfer` は、対象人物の同一性をできるだけ保ちながら、人物写真のヘアスタイルを変更するための AI agent Skill です。

この Skill は 2 枚の人物画像を直接ブレンドするのではなく、より安全な 2 段階のワークフローを使います。

1. ヘアスタイル参考画像から、髪型と髪色をテキストとして抽出します。
2. そのテキストを構造化された JSON ワークフローに適用し、対象人物の髪だけを編集します。

目的は、対象画像の顔、表情、メイク、服装、背景、全体的な人物らしさを保ちながら、参考画像のヘアスタイルと髪色だけを反映することです。

## 特長

- `REFERENCE_0` を唯一の人物同一性ソースとして使用します。
- `REFERENCE_1` は髪型と髪色の分析にのみ使用します。
- 参考人物の顔、メイク、年齢感、雰囲気、同一性が対象人物へ移ることを避けます。
- 最終的に 1 枚の左右結合画像を生成します。
  - 左側：新しい髪型の正面ビュー
  - 右側：同じ髪型の背面ビュー
- 左上に繁体字中国語の髪型ラベルを追加します。
- 再利用可能な JSON ワークフローテンプレートを含みます。
- ヘアスタイル検討、サロン用プレビュー、AI 支援のビジュアルスタイリングに適しています。

## 仕組み

この Skill は、2 枚の入力画像を別々の役割として扱います。

| 参照 | 目的 |
| --- | --- |
| `REFERENCE_0` | 対象人物のポートレート。顔、同一性、表情、肌質、メイク、服装、構図、背景の唯一のソースです。 |
| `REFERENCE_1` | ヘアスタイル参考画像。髪型の形、長さ、レイヤー、カール、質感、髪色の分析にのみ使います。 |

最終生成では、ヘアスタイル分析から得た `scope` と `hair_color` を構造化ワークフローに入力します。これにより、参考画像の顔の特徴が対象人物の同一性に混ざるリスクを下げます。

## インストール

Skill は次のいずれかの場所に配置できます。

複数プロジェクトで使う場合:

```text
~/.agents/skills/hairstyle-transfer/
```

特定のプロジェクト内だけで使う場合:

```text
<project-root>/.agents/skills/hairstyle-transfer/
```

フォルダには少なくとも次のファイルが必要です。

```text
SKILL.md
templates/base-workflow.json
```

## 使い方

2 枚の画像をアップロードします。

1. 対象人物のポートレート
2. ヘアスタイル参考画像

その後、エージェントに `hairstyle-transfer` Skill の使用を依頼します。例:

```text
Use the hairstyle-transfer Skill to change the first person's hairstyle to match the second image.
```

Skill はまず 2 枚目の画像から髪型と髪色だけを分析し、その結果を JSON ワークフローを通じて 1 枚目の画像へ適用します。

## 出力

最終出力は 1 枚の画像です。

- 左側：新しい髪型になった対象人物の正面ビュー
- 右側：同じ人物と同じ髪型の背面ビュー
- 左上ラベル：繁体字中国語の髪型名または短い説明
- ラベル色：`#721D24`

この Skill は、複数の個別画像を出力しない想定です。

## 設計方針

この Skill は、髪型の完全な一致よりも人物同一性の保持を優先します。

髪型が対象人物の認識可能な顔の特徴を隠したり変形させたりする場合、ワークフローはまず対象人物の同一性を保ち、必要に応じて髪型の変化を弱めます。

ヘアスタイル参考画像を、顔、メイク、年齢感、表情、人物同一性の参考として使ってはいけません。
